- バージョン: 0.1（ドラフト）
- 最終更新: 2026-06-11
- プラットフォーム: Webブラウザ（PC専用）
- デプロイ先: GitHub Pages

## 1. コンセプト

**「勇者がモブで、モンスターが主人公」の立場逆転サバイバー。**

プレイヤーは魔王軍の一兵卒オーク。どこからともなく無限に湧いてくる「勇者」たち（剣士・アーチャー・メイジ・パラディン…）の大群を薙ぎ払いながら、**5分間生き残る**ことが目標。

レベルアップのたびに「肉を喰らう」「雄叫び」「仲間オーク召喚」などオークらしいスキルを3択から選び、自分だけのビルドを組む。ヴァンパイアサバイバー系の中毒性あるループを、オークの世界観とネタで包む。

### 1.1 なぜバズるか（バズ導線）

| 要素 | 内容 |
|---|---|
| 逆転ネタ | 「勇者が無限に湧くんだが？」という一言で伝わるフック。SNSで語りたくなる |
| 共有カード | リザルト画面で「討伐した勇者: 1,234人」の画像カードを生成 → ワンクリックでX(Twitter)共有 |
| 称号システム | 討伐数・生存時間に応じて称号付与（「村の小悪党」→「王国の悪夢」→「勇者卸売業者」）。称号が共有カードに載る |
| 即プレイ | URLを開いて3秒でロード完了、タイトル画面からワンキーで開始。説明不要の操作（移動だけ） |
| 5分完結 | 「ちょっと1回だけ」が成立する長さ。負けても即リトライ |

### 1.2 ターゲット

- SNS（主にX）でゲームネタを見る20〜40代
- ヴァンパイアサバイバー系の文法を知っている層（説明コスト最小）
- 知らない層にも「動くだけ・攻撃は自動」で遊べる

## 2. ゲームルール

### 2.1 基本ループ

```
タイトル → ゲーム開始 → [移動して敵を倒す → 経験値取得 → レベルアップで3択スキル選択] のループ
→ 5分生存で勝利 / HP 0 で敗北 → リザルト（スコア・称号・共有カード）→ リトライ
```

- **勝利条件**: 5分間（300秒）生存。5:00時点で最終ボス「真の勇者」が出現し、これを討伐してクリア
- **敗北条件**: HPが0になる
- **1プレイ時間**: 最大約6分（5分＋ボス戦）。早期敗北なら数分

### 2.2 スコア

- スコア = 討伐数ベース。敵種別に係数（見習い剣士 1pt 〜 パラディン 20pt、ボス 500pt）
- リザルトに表示: 生存時間 / 討伐した勇者の総数 / 最大レベル / 取得ビルド / 称号
- ハイスコアは `localStorage` に保存（サーバ不要）

### 2.3 経験値とレベルアップ

- 敵を倒すと経験値ジェム（緑の魂）をドロップ。近づくと自動回収（吸引半径あり）
- レベルアップ時にゲームを一時停止し、スキル3択を表示
- 必要経験値はレベルごとに逓増: `必要EXP(n) = 3 + n * 6`（実装時にバランス調整）

## 3. プレイヤー（オーク）

### 3.1 操作（PC専用）

| 入力 | 動作 |
|---|---|
| WASD / 矢印キー | 8方向移動 |
| マウス | 不使用（メニュー選択のみクリック可。1〜3キーでも選択可能） |
| ESC / P | ポーズ |

**攻撃はすべて自動。** プレイヤーの意思決定は「位置取り」と「ビルド選択」のみ。

### 3.2 基本ステータス（初期値）

| 項目 | 値 |
|---|---|
| HP | 100 |
| 移動速度 | 5.0 unit/s |
| 初期武器 | オークの棍棒 |
| ジェム吸引半径 | 2.0 unit |

### 3.3 カメラ

- 見下ろし型（クォータービュー、俯角約60度）の固定追従カメラ
- プレイヤーを常に画面中央に配置

## 4. 武器・スキル

レベルアップごとに3択。**武器（アクティブ）最大4枠 + パッシブ最大4枠**。各スキルはLv5まで強化可能。同じスキルが候補に出たらレベルアップ扱い。

### 4.1 武器（アクティブ・自動発動）

| 名前 | 効果 | 強化方向 |
|---|---|---|
| オークの棍棒（初期） | 前方を周期的に薙ぎ払う近接攻撃 | 範囲・威力・速度 |
| 骨投げ | 最寄りの敵へ骨を投擲。貫通あり | 投擲数・貫通数 |
| 地団駄 | 自分中心の円形衝撃波。ノックバック付き | 半径・頻度 |
| 雄叫び | 扇状範囲の敵を怯ませ＋ダメージ | 範囲角・怯み時間 |
| ブタ突進 | 野生のブタが画面を横切り敵を轢く | ブタの数・サイズ |
| 仲間オーク召喚 | 自動で戦う子分オークを召喚（最大3体） | 体数・子分の強さ |

### 4.2 パッシブ

| 名前 | 効果 |
|---|---|
| 筋肉 | 全攻撃力 +10%/Lv |
| 分厚い皮膚 | 被ダメージ -8%/Lv |
| 豚足 | 移動速度 +8%/Lv |
| 大食い | 敵がたまに肉をドロップ。肉の回復量 +20%/Lv |
| でかい図体 | 最大HP +20%/Lv（見た目も少し大きくなる） |
| 鼻が利く | ジェム吸引半径 +30%/Lv |

### 4.3 回復

- 通常回復手段はドロップする「肉」のみ（基本回復量 20）
- 「大食い」パッシブでドロップ率・回復量が向上

> ネタとしての面白さ優先。武器名・演出は「オークっぽさ」を最重要視する。武器進化システム（武器Lv MAX + 対応パッシブで超進化）は v2 で検討。

## 5. 敵（勇者たち）

### 5.1 敵タイプ

| 名前 | 行動 | HP係数 | 速度 | スコア |
|---|---|---|---|---|
| 見習い剣士 | プレイヤーへ直進・接触ダメージ | 1 | 遅 | 1 |
| 冒険者（量産型） | 直進。数で押す | 2 | 中 | 2 |
| アーチャー | 一定距離を保ち矢を射る | 2 | 中 | 5 |
| メイジ | 遠距離から魔法弾（遅い・誘導なし） | 3 | 遅 | 8 |
| ナイト | 高HP・ノックバック耐性 | 8 | 遅 | 10 |
| パラディン（エリート） | 高HP・周囲の勇者を加速するオーラ | 25 | 中 | 20 |
| 真の勇者（最終ボス） | 5:00出現。突進・剣戟の状態機械 | 180 | 速 | 500 |

### 5.2 スポーンタイムライン（5分構成）

| 時間 | イベント |
|---|---|
| 0:00〜 | 見習い剣士がパラパラと出現 |
| 0:30〜 | 冒険者追加。出現レート上昇 |
| 1:00〜 | アーチャー追加 |
| 1:30 | **第1波**: 剣士の大群が円形に包囲出現 |
| 2:00〜 | メイジ追加 |
| 2:30〜 | ナイト追加。BGM転調 |
| 3:00 | **第2波**: パラディン1体＋取り巻きの中隊 |
| 3:45〜 | 全種混成。出現レート最大へ漸増 |
| 4:30 | **第3波**: パラディン2体の親衛隊 |
| 5:00 | 雑魚スポーン停止 → **最終ボス「真の勇者」出現**。討伐でクリア＝エンディングシネマへ |

- 敵HPは経過時間でスケール: `HP = 基礎HP × (1 + 経過分 × 0.3)`（5分で2.5倍）
- 同時存在上限: 300体（超過時は最古の画面外の敵をリサイクル）

## 6. UI / UX

### 6.1 画面一覧

1. **タイトル**: ロゴ＋「SPACEで開始」＋ハイスコア表示。装飾最小、即開始重視
2. **ゲーム中HUD**: 上部に残り時間（カウントダウン）、左上にHPバー・レベル・経験値バー、右上に討伐数
3. **レベルアップ選択**: 時停止＋3択カード（1/2/3キー or クリック）
4. **ポーズ**: 再開 / リトライ / タイトルへ
5. **リザルト**: 勝敗演出 → スコア詳細 → 称号 → **共有カード画像**＋「Xで共有」ボタン → リトライ

### 6.2 共有カード（バズの要）

- Canvas 2D でリザルト画像を動的生成（1200×630、OGP比率）
- 記載内容: タイトルロゴ / 称号 / 討伐した勇者: N人 / 生存時間 / ビルドのアイコン
- 「Xで共有」= Web Intent URL（テキスト＋ゲームURL）。画像はユーザーがダウンロード→添付、またはクリップボードコピー（`navigator.clipboard`）

### 6.3 称号テーブル（討伐数ベース・抜粋）

| 討伐数 | 称号 |
|---|---|
| 〜99 | 村の小悪党 |
| 〜499 | 街道の脅威 |
| 〜999 | 砦の主 |
| 〜1999 | 王国の悪夢 |
| 2000〜 | 勇者卸売業者 |

## 7. ビジュアル / オーディオ

### 7.1 ビジュアル方針

- **ローポリ＋フラットシェーディング（トゥーン調）**。アセット軽量・大量描画・スタイル統一の3点で有利
- プレイヤーオーク: 緑肌・ガタイの良いデフォルメ体型。やられ役ではなく「主役」の愛嬌を持たせる
- 勇者たち: 色違いベースの量産デザイン（InstancedMesh 前提でモデル共通化）
- 地形: 草原＋まばらな木・岩のシンプルな平原（境界なしの見た目、実際は移動範囲制限あり）
- ダメージ数字ポップ、討伐時の魂エフェクトなど「気持ちよさ」の演出を優先投資

### 7.2 アセット調達

- 3Dモデル: 自作（プリミティブ組み合わせ）または CC0 素材（Quaternius, Kenney 等）
- BGM/SE: CC0/フリー素材（ライセンス表記をクレジットに記載）。SEはWeb Audio APIで再生

## 8. 技術仕様

### 8.1 スタック

| 項目 | 採用技術 | 理由 |
|---|---|---|
| 言語 | TypeScript | 型安全・リファクタ耐性 |
| 3D描画 | Three.js | ブラウザ3Dのデファクト。情報量が多い |
| ビルド | Vite | 高速・GitHub Pages向け静的出力が容易 |
| 物理 | 使わない | 距離ベースの自前円判定で十分。負荷とサイズを削減 |
| 状態管理 | 自前の軽量ゲームループ（シーン＋エンティティ） | フレームワーク不要 |
| 永続化 | localStorage | ハイスコア・設定のみ。サーバレス |
| テスト | Vitest | ロジック層（ダメージ計算・スポーン・レベルテーブル）の単体テスト |
| CI/CD | GitHub Actions → GitHub Pages | push で自動デプロイ |

### 8.2 アーキテクチャ方針

- **描画とロジックを分離**: ゲームロジック（座標・HP・スポーン・衝突）は Three.js 非依存の純粋TSモジュールにする → Vitest で実機能をテスト可能にする
- 敵・ジェム・弾は **InstancedMesh** で描画（タイプごとに1ドローコール）
- 衝突判定は **均一グリッドの空間分割**（セルサイズ≒最大敵半径×2）で O(n) 近似
- 敵・ジェムは**オブジェクトプール**で生成/破棄ゼロ運用（GCスパイク回避）
- 乱数はシード可能な自前PRNG（将来のデイリーチャレンジ・リプレイ検証に備える）

### 8.3 パフォーマンス目標

| 指標 | 目標 |
|---|---|
| FPS | 60fps（ミドルレンジノートPC、敵300体時） |
| 初回ロード | 転送合計 < 3MB、3秒以内にタイトル表示 |
| ドローコール | < 50 |

### 8.4 対応ブラウザ

- Chrome / Edge / Firefox / Safari の最新版（PC）
- WebGL2 必須。非対応時はメッセージ表示

## 9. スコープ定義

### MVP（まず遊べる縦切り）

- プレイヤー移動＋棍棒（初期武器）＋見習い剣士1種
- 経験値→レベルアップ→3択（武器2種＋パッシブ2種程度）
- 5分タイマー＋敗北/勝利（ボスなし、生存のみ）
- 仮ジオメトリ（箱・カプセル）でよい

### v1.0（公開版）

- 武器6種・パッシブ6種・敵7種（ボス含む）フル実装
- スポーンタイムライン全実装
- ローポリモデル・エフェクト・BGM/SE
- タイトル/リザルト/共有カード/称号
- GitHub Pages 公開＋OGP設定

### v2（公開後の反応次第）

- 武器進化システム
- デイリーチャレンジ（シード固定）
- スマホ対応（仮想スティック）
- オンラインランキング（要サーバ検討）

## 10. リスクと対策

| リスク | 対策 |
|---|---|
| 敵300体で性能不足 | InstancedMesh＋空間分割＋プールを初期から導入。MVP時点で300体負荷試験を行う |
| サバイバー系の既視感 | オークのネタスキル・称号・共有カードで「語られる差別化」を作る |
| バランス崩壊（強すぎ/弱すぎビルド） | ロジック層を分離しシミュレーション（自動プレイ）でDPS検証可能にする |
| アセット制作が膨らむ | 量産敵はモデル共通・色違い。MVPは仮ジオメトリで進め、見た目は最後に差し替え |
